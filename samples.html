Writing Samples

In my profession, I have been responsible for designing and writing quality documentation in support of various software and hardware products, ranging from advanced technical users such as engineers and developers to more end-user friendly audience groups. Documentation is also a highly collaborative process and much of my work involved information gathering from across multiple Subject Matter Experts (SME) and product owners to successfully understand and manage document requirements and expectations. 

Across my career I have become well-versed in many DITA XML markup editors such as Oxygen and Author-It. Refer to my Resume for a complete listing.

The samples shown in this portfolio highlight document types I have produced and showcase my documentation approach, design, and writing abilities. I have primarily written for IT software and have a background in basic JAVA programming.

Configuration Guide

This configuration guide was created for runtime configuration without using a user interface wizard. Writing this document required close correspondence with developers to create clear procedural instructions for users.

User Guide

This guide was intended to help non-technical users (merchants) track and reconcile funds they received during payout settlement, which was compiled in .csv format file called the Settlement Report. I worked closely with the finance and support team to create a guide that would address merchant concerns in reading and understanding a highly complex .csv report.
